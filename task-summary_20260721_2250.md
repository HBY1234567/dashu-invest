# 2026-07-21 22:50 - 修复持仓卡片按钮不显示

## 问题
用户截图反馈：模拟炒股页面的 3 个持仓卡片（上港集团/中原高速/中国石化）依然没有"加仓/卖出"按钮。

## 根因
`renderPortfolio` 函数里有个错误判断：
```js
if(holdings.length>0){
  html+='<div class="holding-actions">...按钮...</div>';
}
```

`holdings` 是函数局部变量，等于 `tradingState.holdings`（模拟账户）。但用户的持仓数据其实在 `state.practice`（老实践表）里——`tradingState.holdings` 是空的。所以 `holdings.length>0` 为 false，按钮就被跳过了。

数据流：syncPracticeFromHoldings() 是单向同步（从 tradingState 同步到 practice），不会反向。所以老实践表里的数据没有自动进入模拟账户。

## 修复方案（commit 7fa2b43 + efbf855）

### 1. 移除 `if(holdings.length>0)` 判断
- 改为始终显示按钮
- 用 `isInTrading = tradingState.holdings.some(h => h.code === item.code)` 判断当前股票是否在模拟账户
- 如果在：显示"加仓" + "卖出"按钮
- 如果不在：只显示"买入"按钮（点击后新建持仓）

### 2. 加 `migratePracticeToTrading()` 迁移函数
- 在 `loadTradingState()` 之后立即调用
- 仅当 `tradingState.holdings` 为空 + `state.practice` 有数据时执行
- 把 practice 里的每条记录复制到 tradingState.holdings
- 同步设置 firstBuyPrice/firstBuyShares（保证加仓信号能正常触发）
- 重新计算可用资金：INIT_BALANCE - 总持仓成本 - 1000（留点余量）

### 3. 顺手清理
- 删除了意外提交进仓库的 `index.html.backup-20260721` 和 `task-summary_20260720_1858.md`

## 用户操作
刷新浏览器（建议下拉刷新或无痕模式）后，3 个持仓卡片底部会出现：
- **加仓** 按钮（橙色"买入"色，调用 `showBuyConfirmModal(code, true)`，弹窗预填首次股数）
- **卖出** 按钮（红色，调用 `showSellSignalModal(code)`）

如果 52 周位置 >50%，会显示一个橙色"卖一半（点此操作）"的可点击信号条；>80% 显示红色"清仓（点此操作）"。
